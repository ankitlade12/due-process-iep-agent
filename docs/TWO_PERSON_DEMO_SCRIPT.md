# Two-person demo script (2:40 target)

Use synthetic/de-identified data only. Rehearse once with live Qwen and record a
backup take with the local path. Never expose a terminal containing `.env`.

## 0:00–0:25 — Presenter A: the record problem

“An IEP can promise a service three times each week, but the evidence of delivery
lives in months of inconsistent logs. Due Process reconstructs that record while
keeping consequential math outside the model.” Show the upload screen and sample
redacted inputs.

## 0:25–1:05 — Presenter B: Qwen plus deterministic core

Run **Live Qwen review**. Show the task provenance panel. “This reports successful
calls, latency, model, request ID when returned, and any safe fallback. Qwen reads
messy language. The ledger is deterministic.” Show promised, delivered, excused,
and unexcused minutes.

## 1:05–1:40 — Presenter A: evidence chain

Open one review signal. Show the IEP excerpt, exact log IDs, authorities, and
packet download. “The 15% setting is a configurable screening policy—not a legal
bright line. The gate rejects a claim without its evidence IDs.”

## 1:40–2:10 — Presenter B: human and external tool

Show the held draft and approval checkpoint. Cut to the pre-run authenticated
Function Compute result and OSS receipt. “Nothing external happens until approval.
The stored packet is content-addressed and returns a SHA-256 audit receipt.”

## 2:10–2:30 — Presenter A: proof

Show the offline evaluation output, passing GitHub Actions check, public repository,
and Alibaba Workbench Function Compute page. State the benchmark limitation:
“Most cases are synthetic; this proves software consistency, not legal outcomes.”

## 2:30–2:40 — Both: close

“Receipts over rhetoric: bounded Qwen, reproducible evidence, and a human in
control.”
